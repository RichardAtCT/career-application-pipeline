# Workflow

## 1. Intake

The user provides a CV/resume and optionally target preferences. The agent creates a local workspace and extracts CV text.

## 2. Candidate profile

The agent converts the CV into structured facts and unknowns. The user reviews the profile before the search is treated as authoritative.

## 3. CV and cover-letter improvement

The agent produces:

- `cv/cv-feedback.md` with suggested edits and missing metrics;
- `cv/example-cover-letter.md` to establish tone and positioning.

## 4. Search design

The agent creates search tracks. A strong workflow separates different directions rather than mixing them into one generic search.

Examples: product leadership, AI strategy, operations leadership, data/analytics, public sector/nonprofit, fractional/advisory, startup operator.

## 5. Source inventory

The agent creates a source inventory, prioritizing direct sources and ATS endpoints. Search engines are useful backfill, not the only source.

## 6. First manual search

Before automation, run a manual search, verify jobs, score them, update the tracker, and show results. This calibrates the pipeline.

## 7. Application pack

For each strong role, generate a pack containing the job description, source URLs, fit memo, company brief, tailored CV, cover letter, suggested answers, and provenance.

## 8. Automation

After calibration, set up a strong-match monitor and weekly digest using the user's preferred scheduler or agent runtime.
